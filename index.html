Message types

407
Auth challenge407 Proxy Auth Required — proxy demands credentials before forwarding. Triggers ACK + retry.
INV
INVITEInitiates or modifies a session. Carries SDP offer (codec, IP, port).
1xx
Provisional (1xx)100 Trying = suppress retransmit (proxy-generated, never forwarded). 180 Ringing = Bob's phone is alerting, forwarded to Alice. 183 Session Progress = early media.
200
200 OKFinal success. When replying to INVITE, carries SDP answer (Bob's codec + RTP port).
ACK
ACKConfirms 200 OK for INVITE. Completes 3-way handshake. Not used for other methods.
BYE
BYETerminates an active dialog. Either party can initiate. Follows Record-Route path.

Key headers in this flow

Via
ViaAdded by each hop. Response is routed back via Via headers in reverse order.
R-R
Record-RouteProxies inject themselves so they remain in the signaling path for BYE.
SDP
SDP bodyINVITE carries offer (Alice's codec list, IP, port). 200 OK carries answer (Bob's choice).
PA
Proxy-AuthorizationCredential header Alice adds in F4 after receiving the 407 challenge (MD5 digest).
CID
Call-IDUnique string linking all 23 messages to the same session. Set in F1, never changes.
CSeq
CSeqSequence number + method. F4 has higher CSeq than F1 — incremented on retry.

3 phases at a glance

①
Call Setup (F1–F17)Auth handshake → INVITE chain through both proxies → provisional 100/180 → 200 OK with SDP → ACK to establish dialog.
②
RTP MediaReal-time UDP voice/video. Flows directly between Alice and Bob — the SIP proxies are completely bypassed.
③
Teardown (F18–F23)BYE traverses the proxy chain (per Record-Route). Each hop releases state and replies 200 OK.

Provisional Responses — 100 Trying vs 180 Ringing

Property 100 Trying 180 Ringing
Sent by Proxy (each hop locally) End device — Bob
Meaning "I received your request, processing" "Bob's phone is physically ringing"
Forwarded by proxies No — never forwarded upstream Yes — relayed all the way to Alice
Effect on sender's neighbor Stops retransmission timer Alice hears ringback tone
Contains Contact header No Yes — Bob's direct address
Introduces Bob's tag No Yes — To: tag=bb88 appears here first
Changes dialog state No Yes — dialog moves to Early state
In this flow F6 (P1→Alice), F8 (P2→P1) F9 (Bob→P2), F10 (P2→P1), F11 (P1→Alice)

SIP Response Codes Reference

Code Name Meaning & When You See It
100 Trying Proxy acknowledges receipt. Stops retransmission timer. Never forwarded upstream.
180 Ringing Called party's device is alerting the user. Phone is ringing. Forwarded to caller.
181 Call Is Being Forwarded UAS is forwarding the call to another destination.
182 Queued Called party is temporarily unavailable — call is queued.
183 Session Progress Used for early media (e.g. ringback tone sent as RTP before answer).
200 OK Request succeeded. On INVITE: carries SDP answer, dialog established.
202 Accepted Used for REFER and SUBSCRIBE — request accepted but not yet acted upon.
301 Moved Permanently Contact has a new permanent address. Caller should retry there.
302 Moved Temporarily Contact is temporarily at a different address. Common in call-forwarding.
400 Bad Request Malformed syntax — the server could not understand the request.
401 Unauthorized End-device (UAS) demands authentication. Uses WWW-Authenticate header (not Proxy-Authenticate).
403 Forbidden Server understood the request but refuses. Credentials valid but not authorized.
404 Not Found Called party does not exist at this domain. URI is unknown.
407 Proxy Auth Required Proxy demands credentials. Seen in F2 — triggers ACK (F3) + retry with Proxy-Authorization (F4).
408 Request Timeout No response received in time. Server gave up waiting.
480 Temporarily Unavailable Called party exists but is not available right now (offline, DND).
481 Call/Transaction Does Not Exist Received BYE or CANCEL for a dialog the server has no record of.
486 Busy Here Called party is busy on another call at this specific device.
487 Request Terminated INVITE was cancelled by a CANCEL request before being answered.
488 Not Acceptable Here SDP offer was rejected — no codec or media format overlap between the two sides.
500 Internal Server Error Generic server-side failure. Something went wrong on the proxy or UAS.
503 Service Unavailable Server is overloaded or down for maintenance. Often includes Retry-After header.
504 Server Timeout Proxy did not receive a response from the downstream server in time.
513 Message Too Large Request body or headers exceed the server's maximum size.
600 Busy Everywhere Called party is busy on all registered devices — not just one.
603 Decline Called party explicitly rejected the call. User pressed "Decline".
604 Does Not Exist Anywhere Called party is known to not exist — not just unknown at this domain.
