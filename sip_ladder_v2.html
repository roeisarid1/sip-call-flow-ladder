Message types

407
Auth challenge407 Proxy Auth Required — proxy demands credentials before forwarding. Triggers ACK + retry.
INV
INVITEInitiates or modifies a session. Carries SDP offer (codec, IP, port).
1xx
Provisional (1xx)100 Trying = suppress retransmit. 180 Ringing = remote side is alerting user.
200
200 OKFinal success. When replying to INVITE, carries SDP answer (Bob's codec + RTP port).
ACK
ACKConfirms 200 OK for INVITE. Completes 3-way handshake. Not used for other methods.
BYE
BYETerminates an active dialog. Either party can initiate. Follows Record-Route path.

Key headers in this flow

Via
ViaAdded by each hop. Response is routed back via Via headers in reverse order.
R-R
Record-RouteProxies inject themselves so they remain in the signaling path for BYE.
SDP
SDP bodyINVITE carries offer (Alice's codec list, IP, port). 200 OK carries answer (Bob's choice).
PA
Proxy-AuthorizationCredential header Alice adds in F4 after receiving the 407 challenge (MD5 digest).
CID
Call-IDUnique string linking all 23 messages to the same session. Set in F1, never changes.
CSeq
CSeqSequence number + method. F4 has higher CSeq than F1 — incremented on retry.

3 phases at a glance

①
Call Setup (F1–F17)Auth handshake → INVITE chain through both proxies → provisional 100/180 → 200 OK with SDP → ACK to establish dialog.
②
RTP MediaReal-time UDP voice/video. Flows directly between Alice and Bob — the SIP proxies are completely bypassed.
③
Teardown (F18–F23)BYE traverses the proxy chain (per Record-Route). Each hop releases state and replies 200 OK.
